- (decision) Design lead = Linear: dark-first, dense, focused, indigo/violet accent, keyboard-forward, muted grays, low chrome. Stripe/Vercel/Notion may influence details but Linear governs.
- (decision) Color mode = Both (system-aware) light+dark, follows OS preference with a manual toggle. Current app is light-only; dark becomes first-class.
- (decision) Implementation approach = Adopt Tailwind CSS for utility styling + a small set of hand-built reusable React components (Card, Table, Badge, Button, Input, Dialog, etc). Replaces the current hand-rolled plain-CSS approach. One build-tool dependency added; no headless component library.
- (decision) Charts scope = Dashboards only (role dashboards get charts: balance usage, pending counts, team-on-leave). Reports stay tables + CSV as today. Charts are NET-NEW (original build deliberately excluded them: SM-C2).
- (assumption) HARD CONSTRAINT (from brief): redesign is frontend-only. No change to backend, API contracts, business logic, auth/authz, or DB. All ~18 existing role-gated panels and their react-query hooks are preserved; work is layout + visual + component-extraction + routing.
- (decision) IA = group-by-function sidebar. Sections: Overview (Dashboard), Leave (Request Leave, My Leave, Cancellations), Team·Manager (Approvals[+dept calendar at decision], My Team), Reports·Mgr/Admin (Leave Report+CSV), Administration·Admin (Cancellation Requests, Employees, Departments, Leave Types, Holidays, Policy Changes, Review Flags, Audit Log), Account (Notifications, Profile). Items hidden unless role can reach them; the existing per-panel useMe self-gate stays the source of truth.
- (decision) Dashboard = ONE adaptive surface. Composes role panels (Employee balances always; Manager queue-summary and Admin org-totals added by role). New charts live here. It is the post-login landing page.
- (decision) Responsive nav = collapsible sidebar: full labeled rail on desktop, icon-only rail on medium, hamburger-triggered slide-in drawer on narrow. One nav model across breakpoints. Department leave calendar (Story 3.3) stays embedded in Approvals at decision time, not a standalone nav item.
- (decision) Voice = PRESERVE the existing precise, plain, no-marketing voice; present it in cleaner UI. Do not rewrite functional copy. (Note for microcopy section: existing implementation-detail asides like the Bearer-token explainer may move to less prominent placement but wording is preserved.)
- (decision) Command palette (Cmd-K) = SKIP for this redesign. Scope stays limited to restyling + routing existing surfaces. Candidate for a later enhancement.
- (decision) Accessibility floor = same as original project stance: legible contrast in BOTH light+dark themes, fully keyboard navigable, visible focus rings, semantic tables/forms. No formal WCAG AA commitment (WCAG was a declared non-goal). Focus-visible rings become more important now that dark mode exists.
- (assumption) Charts library = Recharts (React, lightweight, composable) assumed for the dashboard charts, pending confirmation at build time. Alternatives: visx, or hand-rolled SVG. Keep charts minimal/Linear-flat: no 3D, no heavy gradients.
- (decision) Loading states = skeleton loaders (Linear-style shimmer) for cards/tables replacing today's text 'loading...'; empty states get an icon+message+primary action; errors get an inline retry affordance. Preserve existing react-query isPending/isError branch points.
